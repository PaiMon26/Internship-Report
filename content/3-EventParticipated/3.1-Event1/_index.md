# Summary Report: "AWS re:Invent Recap Vietnam 2025"

### Event Objectives

- Summarize and share standout technologies from AWS re:Invent 2025
- Introduce practical AI and cloud development trends
- Provide a perspective on how modern AI systems are built
- Help attendees quickly approach newly released AWS services

### Key Highlights

#### 1. LLM usage patterns

**Quick Research**

- Analyze complex data
- Generate reports quickly

Used when:

- Data volume is large
- The business needs fast insights

**Quick Flows**

- Build multi-step pipelines
- Summarize emails
- Aggregate data
- Analyze and evaluate information

The LLM acts as a workflow engine.

**Quick Automate**

- Handle complex problems
- Combine multiple modules

Development direction:

- AI automation systems replacing traditional backend logic in some workflows

#### 2. AWS AI Stack

**Amazon Bedrock**

- Managed platform for running LLMs
- Supports multiple foundation models

**Bedrock AgentCore**

- Build AI agent systems
- Orchestrate system logic

#### 3. Components in an AI System

**Policy**

- Control model output
- Ensure safety and governance

**Evaluation**

- Evaluate models
- Accuracy
- Reliability

**Enhancements**

- Personalization
- Real-time processing
- User authentication and differentiation

**Reinforcement Fine-tuning**

- Improve models based on feedback

**Strands Agent**

- Framework support for TypeScript and Python

**Nova Model**

- Multiple versions such as Lite and upgraded variants

#### 4. Data Layer

**Embedding**

- Applied to text
- Applied to images
- Applied to video

Used for:

- Semantic search
- RAG (Retrieval-Augmented Generation)

**Search Web**

- Retrieve data from the internet

**Force Mode**

- The model learns from user behavior
- The model learns from real-world data

**Action Layer (Act)**

- Execute actions
- Call APIs
- Interact with systems

**SageMaker AI**

- Train models
- Deploy and scale
- Manage pipelines

#### 5. Networking in AWS systems

- Multi-cloud interconnect
- Transit Gateway for connecting multiple VPCs
- Site-to-site VPN
- Direct Connect for more stable connectivity than the public internet

Comparison:

- VPN depends on internet quality
- Direct Connect is more stable

Note:

- Costs increase by region and gateway

#### 6. Infrastructure

**Container-based**

- ECS

**Serverless**

- AWS Lambda

#### 7. MCP (Model Context Protocol)

- A standard for connecting models and tools
- A standard for connecting models and external systems

Use cases:

- Agents calling tools through MCP
- Connecting clients, AWS, and other systems

#### 8. Agent Architecture

- Multi-agent systems

Types of agents:

- Planner
- Tool
- Executor

**Agent + Memory**

- Store conversation history
- Preserve context

**Serverless memory**

- No need to build the infrastructure manually

#### 9. Storage & Logging

Store:

- Conversation history
- Context

Support:

- Debugging
- Model improvement

#### 10. Key Takeaways

A modern AI system includes:

- Model (Bedrock)
- Agent (AgentCore)
- Data (Embedding, Search)
- Tooling (MCP)
- Infrastructure (ECS, Lambda)
- Networking

Major trends:

- Agent-based architecture
- Multi-agent orchestration
- LLMs as the core engine of the system

### Key Learnings

#### Technology Mindset

- AI is not just a chatbot, but a complete system
- LLMs can play a central role in the system
- I now better understand modern AI architecture on the cloud

#### Technical Knowledge

I became familiar with:

- AI agents
- Embedding
- RAG
- Multi-agent systems

I also gained a basic understanding of how these AI components connect together.

#### Applying to Work

Possible applications:

- Using AI to automate tasks
- Understanding how to integrate AI into systems

In the future:

- Building AI-based systems
- Applying agent-based architecture

### Event Experience

The "AWS re:Invent Recap Vietnam 2025" event provided a broad view of new technologies in AI and Cloud, especially approaches to building modern AI systems on AWS.

The sessions on AI agents, embeddings, and multi-agent systems expanded the perspective on AI as a complete system rather than a standalone chatbot experience. Some topics were still relatively new, so much of the content was absorbed at a high level.

Overall, the event helped clarify current technology trends and practical directions for applying AI and Cloud in real systems.

### Lessons Learned

- AI systems require many components, not just a model
- Agent-based architecture is an important trend
- Cloud plays a foundational role in AI systems
- I need to learn from basics to advanced topics to understand the field more deeply
